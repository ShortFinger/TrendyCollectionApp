# 抽卡机页（`/pages/card/index`）线框结构 — 设计说明

**日期:** 2026-04-13  
**状态:** 已定稿（待实现计划）

## 背景与目标

抽卡机落地页需与产品线框在 **信息架构与 DOM 分区** 上对齐：**系统导航栏 + 主区（左封面、右侧双入口）+ 底栏四连抽**。本阶段 **只要求结构一致**，**不**以视觉还原为验收标准（颜色、间距、圆角等可在后续迭代细化）。

## 范围

- **包含:** `TrendyCollectionApp/pages/card/index.vue` 在 **活动可用且数据加载成功** 时的页面骨架；与现有 **加载中 / 失败 / 不可用** 三种状态兼容（这三种状态 **不** 强制套用线框主区+底栏）。
- **包含:** 顶栏采用 **系统默认导航**（`pages.json`，**非** `navigationStyle: custom`），与线框中「返回 + 标题」由系统栏承担；可选在数据就绪后用 `uni.setNavigationBarTitle` 将标题设为活动标题（与现有 VO 一致）。
- **不包含:** 抽卡业务接口、支付、动画、真实概率与说明内容；侧链与四钮的 **具体** 跳转目标与文案规则（除占位外）。
- **不包含:** 子组件拆分（首版单页内 flex 分区即可，见关联 spec [活动落地页](./2026-04-13-activity-landing-pages-by-type-design.md) 中对 `card` 壳页的描述）。

## 线框对应关系

| 线框区域 | 实现方式 |
|----------|----------|
| 顶栏（左返回、中标题） | **系统导航栏**；`pages.json` 已配置 `navigationBarTitleText`（如「抽卡机」）；动态标题可选。 |
| 主区背景 | 成功态最外层主内容区容器（浅灰占位类名即可，具体样式非本阶段验收项）。 |
| 中央「封面」 | 主区内 **左侧** 大块区域：承载封面图/占位（可延续现有 `coverUrl` 数据绑定，仅结构占位亦可）。 |
| 右侧「中赏概率」「购买说明」 | 主区内 **右侧** 竖排 **两个** 可点击区域，文案固定为上述八字。 |
| 底部「1抽 / 5抽 / 10抽 / 20抽」 | **固定底栏** 内 **一行四个** 并列操作区，文案与线框一致。 |

## 布局方案（已定稿：方案 1）

采用 **单页 flex 分区**，不引入 `scroll-view`（除非后续内容增高需要），不拆分子组件。

**成功态结构（自上而下逻辑）：**

1. **主内容区**（纵向 `flex: 1`，占满导航栏以下、底栏以上的空间）：  
   - 内部 **横向一行**：  
     - **左列**：封面区（对应线框中央竖向大矩形）。  
     - **右列**：侧栏，含「中赏概率」「购买说明」两个区块（自上而下）。  
2. **底栏**：贴屏幕底部，横向排列四个抽数入口；需为 **安全区** 预留空间（与项目内其他固定底栏页一致，实现时用 `env(safe-area-inset-bottom)` 等）。

## 数据与路由

- **入参:** 继续 **`activityId`**（query），缺失时行为与现网一致（提示并返回）。  
- **展示数据:** 继续 **`fetchActivityDisplayById` / `display-batch`** 口径与 **`ensureCanonicalActivityRoute`**；`CARD` 类型校验不变。  
- **标题:** 导航栏标题优先与 **`ActivityDisplayVO.title`** 对齐时，在 **`vo` 就绪且可用** 后调用 `setNavigationBarTitle`；若失败则保留 `pages.json` 默认文案。

## 交互（占位）

以下操作在结构阶段 **仅需可点**，行为可为 **toast（如「玩法接入中」）** 或空函数，与当前单 CTA「开始抽卡」占位策略一致：

- 「中赏概率」「购买说明」  
- 「1抽」「5抽」「10抽」「20抽」

## 测试与验收

- **手工:** 带有效 `activityId` 进入页面，可见 **左封面 + 右侧双链 + 底栏四钮**；系统栏显示返回与标题。  
- **状态:** 加载中、错误、活动不可用仍出现原有状态 UI，不强制出现线框主区。  
- **自动化:** 若后续为模板增加快照/单测，在实现计划中约定；本 spec 不强制新增测试文件。
